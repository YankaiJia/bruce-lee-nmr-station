[2]IMAGE REGISTRATION:
[3]Two images are registered, by marking* one or two stars in each image. With two stars the image will also be rotated and resized.
[4]If the pixelformat of the camera is not quadratic, the image has to be scaled before registration with rotation.
[5]When Batch Processing, the size of the selections also affects the search region. The brightest star in the region is choosen as reference.
[6]The program works with floating point values, you dont have to care about rounding errors or too big values.
[7] * Rectangular Selection: Depending on the menu adjustment, a lightness center estimation or a crosscorrelation is performed.
[8]BATCH PROCESSING:
[9]The Start File is the first file from a set. If you combine files (Add to Destination Image), you can specify a Destination File, in which the result will be saved. Without specification it will be saved as "Result.fit".
[10]If the specified file already exists, the files will be combined to it. So you can add additional files to a previous Result, for example.
[11]Without a combining in the Batch, each file will be saved separate. The filename is the name of the Destination File plus a number, or, without a declaration, the original name plus "FW_" at the beginning.
[12]In the adding adjustments, you can specify values for an additional image shift. This way you can add comets or asteroids that dont have enough brightness for a direct selection.
[13]TIPS:
[14]In opposition to the rectangular selections, you can click without moving in the image and the cross will be used as a direct position for registration. This can be useful if an automatic recognition failes.
[15]Prior the calling of a function with PSF, you can mark a star in the image, otherwise a marked star in another image can be taken. At De-/Convolution a whole second image is taken as PSF.
[16]On most functions it is possible to process only a region of the image by making a selection. This can also be used for a fast preview.
[17]With Image Masking you can interpolate dead or hot pixels. All bright pixels in the Mask Image indicates the pixels to be interpolated.
[18]An image will be saved as BMP, PNG or JPG exactly so as shown (with 100% size). Only at Batchprocessing it depends on the real values.
[19]BATCH PROCESSING II:
[20]Without an image combination, each file can also be saved as BMP, PNG, JPG or TIF file. To do that, you have to specify the file extension in the Destination File inputbox. Otherwise it will be saved as Fits file.
[21]If you specify only the extension, the original name plus "FW_" in front of it will be used, with a specified name, it will be used plus a number index. Use a '*' to keep the original file name.
